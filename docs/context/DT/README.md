# Design Thinking Context
## User-Centered Design Artifacts

This folder contains Design Thinking artifacts that capture user needs, behaviors, and pain points discovered through research and empathy exercises.

## Contents

1. **personas.md** - User personas (Student, Staff, Admin)
2. **journey_maps.md** - User journey maps for key workflows
3. **pain_points.md** - Identified user pain points and opportunities
4. **usability_principles.md** - Design principles guiding UI/UX decisions

## Persona-Driven Development

All features should be validated against our three primary personas:
- **Sarah (Student)**: Needs easy booking, clear availability
- **Professor Martinez (Staff)**: Needs approval workflow, resource management
- **Admin Alex**: Needs oversight, analytics, moderation tools
