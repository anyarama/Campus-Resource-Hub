# User Personas
## Campus Resource Hub

### Persona 1: Sarah Chen - The Busy Student
**Role**: Undergraduate Student (Junior, Business Analytics)  
**Age**: 21  
**Tech Savviness**: High

**Goals**:
- Find and book study rooms quickly between classes
- See real-time availability without calling departments
- Collaborate with group project teammates

**Pain Points**:
- Current booking process requires emails/phone calls
- No visibility into room availability
- Double bookings happen frequently
- Can't see what equipment is in each room

**Behaviors**:
- Checks phone constantly between classes
- Prefers mobile-friendly interfaces
- Books resources 1-3 days in advance
- Often needs last-minute bookings

**Quote**: "I just want to see what's available and book it in 30 seconds. Why is it so complicated?"

---

### Persona 2: Professor David Martinez - The Resource Owner
**Role**: Associate Professor & Lab Manager  
**Age**: 42  
**Tech Savviness**: Medium

**Goals**:
- Manage department equipment bookings efficiently
- Approve requests from authorized students only
- Track who is using resources and when
- Reduce email clutter from booking requests

**Pain Points**:
- Email inbox flooded with booking requests
- Hard to track who has borrowed equipment
- No system to enforce equipment return policies
- Can't generate usage reports for department head

**Behaviors**:
- Checks email 2-3 times per day
- Needs desktop interface (rarely on mobile)
- Values reliability over fancy features
- Wants clear audit trails

**Quote**: "I need a system that handles approvals automatically so I can focus on teaching, not managing a calendar."

---

### Persona 3: Alex Kim - The System Administrator
**Role**: IT Administrator, Facilities Management  
**Age**: 35  
**Tech Savviness**: Very High

**Goals**:
- Oversee all campus resource bookings
- Moderate inappropriate content/users
- Generate usage analytics for stakeholders
- Ensure system security and compliance

**Pain Points**:
- No centralized view of all bookings
- Manual moderation takes too much time
- Can't identify problematic users/patterns
- No data for strategic planning

**Behaviors**:
- Monitors dashboard daily
- Needs bulk action capabilities
- Values data visualization
- Security-conscious

**Quote**: "I need visibility into everything happening across campus, with tools to act quickly when issues arise."

---

**How These Personas Inform Design**:
- **Sarah** → Simple mobile UI, quick booking flow, real-time availability
- **Professor Martinez** → Approval queue, email notifications, desktop-optimized
- **Alex** → Admin dashboard, analytics, bulk moderation tools
